# Welcome, New Contributors!

Thank you for your interest in contributing to Player! This guide will help you get started with your first contribution.

## Getting Started

1. **Fork the Repository**
   - Visit the [Player repository on GitHub](https://github.com/player-ui/player)
   - Click the "Fork" button to create your own copy

2. **Clone Your Fork**
   ```bash
   git clone https://github.com/YOUR_USERNAME/player.git
   cd player
   ```

3. **Set Up Development Environment**
   - Follow the [requirements setup](/contributing/#requirements)
   - Install the necessary tools and dependencies

4. **Choose Your Platform**
   - **Core & Web Development**: See the [Core & Web Development Guide](/contributing/core-web)
   - **Android Development**: See the [Android Development Guide](/contributing/android)
   - **iOS Development**: See the [iOS Development Guide](/contributing/ios)

5. **Need Help?**
   - Check the [Troubleshooting Guide](/contributing/troubleshooting) for common issues and solutions

## Making Your First Contribution

### Good First Issues

Look for issues labeled with:
- `good first issue` - Perfect for newcomers
- `help wanted` - Community contributions welcome
- `documentation` - Great for getting familiar with the codebase

### Types of Contributions

- **Bug Fixes**: Fix issues in existing functionality
- **Documentation**: Improve guides, API docs, or examples
- **New Features**: Add new capabilities (discuss in an issue first)
- **Tests**: Add or improve test coverage
- **Examples**: Create new demo applications or examples

### Development Workflow

1. **Create a Branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```

2. **Make Your Changes**
   - Write your code
   - Add or update tests
   - Update documentation if needed

3. **Test Your Changes**
   ```bash
   # Run relevant tests
   bazel test //path/to/your:tests
   
   # For broader testing
   bazel test //...
   ```

4. **Commit Your Changes**
   ```bash
   git add .
   git commit -S -m "feat: add your feature description"
   ```
   
   > Note: The `-S` flag signs your commit, which is required for this project.

5. **Push and Create Pull Request**
   ```bash
   git push origin feature/your-feature-name
   ```
   
   Then create a pull request on GitHub.

## Getting Help

### Community Resources

- **GitHub Issues**: Ask questions or report problems
- **Discussions**: General questions and community chat
- **Pull Request Reviews**: Get feedback on your code

### Code Review Process

1. Automated checks will run on your PR
2. A maintainer will review your code
3. Address any feedback
4. Once approved, your PR will be merged!

### Tips for Success

- **Start Small**: Begin with small, focused changes
- **Ask Questions**: Don't hesitate to ask for help or clarification
- **Be Patient**: Code review takes time, and that's normal
- **Learn from Feedback**: Use review comments to improve your skills

## Understanding the Codebase

### Project Structure

```
player/
├── android/          # Android-specific code
├── ios/             # iOS-specific code  
├── core/            # Core Player functionality
├── plugins/         # Plugin implementations
├── docs/            # Documentation site
└── tools/           # Build tools and utilities
```

### Key Technologies

- **Bazel**: Build system used across all platforms
- **TypeScript/JavaScript**: Core Player logic
- **Swift**: iOS implementation
- **Kotlin**: Android implementation
- **React**: Web implementation
